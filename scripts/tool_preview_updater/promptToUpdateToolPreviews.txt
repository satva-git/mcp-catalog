Refresh catalog tool previews across this repository.

1. If the file does not contain `toolPreview`, skip it without changes.
2. Inspect the MCP server however necessary to determine whether `toolPreview` is stale.

Inspection rules:
- `mcptools` is installed in the runner and available on PATH. Use it when it is a good fit for inspecting MCP servers.
- Prefer live MCP introspection (`tools/list`, negotiated protocol version, remote endpoint, published container image, or runnable server command) when feasible.
- If live introspection is blocked by auth, transport limitations, or unavailable infrastructure, fall back to upstream source code, official docs, README tables, published package metadata, or other authoritative repo-linked sources.
- Use the best available source per server and avoid guessing.

Editing rules:
- Make the smallest correct changes.
- Do not edit YAML files that do not have `toolPreview`.
- Do not change unrelated fields outside `toolPreview` unless absolutely required for preview accuracy.
- Preserve the repo's existing YAML style as much as possible.
- Keep tool descriptions concise and practical.
- Prefer current tool names over deprecated placeholder names.
- If authoritative inspection shows additional current tools that are not present in `toolPreview`, add them.
- If `toolPreview` contains tools that no longer exist in the current server surface, delete them.
- When a preview is stale, refresh the `toolPreview` block against the current tool set rather than only editing already-listed entries.
- Update tool names, tool descriptions, parameter names, and parameter descriptions to match exactly what the server or authoritative documentation provides.
- If tool names, descriptions, parameter names, or parameter descriptions cannot be determined exactly from an authoritative source, make no update for that file.

Repository-level goal:
- Scan all root-level YAML catalog files in one pass.
- If no `toolPreview` updates are needed anywhere, make no code changes.
- If updates are needed, apply them in the working tree.

Before finishing, verify any edited YAML files still parse and that you've inspect every YAML file with tool previews.
